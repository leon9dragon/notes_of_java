## 覆写的定义
- 当父类中的方法与子类所要求的不一样时,  
  在子类中重新定义该方法, 并修改方法体.  
  以达到子类所想要的效果.

## 覆写的原则
1. 实例方法签名必须相同.(参数表和方法名一样)  
   因此改变的只是方法体.
2. 子类方法的返回值类型是和父类方法的返回值相同或者是其子类.
3. 子类方法声明抛出的异常类型和父类方法声明抛出的异常类型相同或者是其子类.
   1. 子类方法中声明抛出的异常小于或等于父类方法声明抛出异常类型.
   2. 子类方法可以同时声明抛出多个属于父类方法声明抛出异常类的子类(RuntimeException 类型除外).
4. 子类方法的访问权限比父类方法访问权限更大或相等.
